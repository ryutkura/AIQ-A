# AIQandA システム概要・使い方ガイド

このディレクトリ（リポジトリ）は、ITエンジニアリングや数学などの質問（QandA）とAIによる解説を蓄積し、Obsidian等のグラフビューで有機的に繋げるためのパーソナルナレッジベースです。
AIエージェントを最大限に活用するための「自動連携システム」が組み込まれています。

---

## 使い方

### 1. QandAの蓄積
AIに質問をすると、AIが自動的に文脈を判断し、適切なフォルダ（`programming`, `math` など）の適切なMarkdownファイルに追記、あるいは新規作成してくれます。
人間が「このファイルに保存して」と細かく指示・管理する必要はありません。全てAIに任せることができます。

### 2. 画像（スクショ）を使った質問
数式やエラー画面など、言葉で説明しづらいものはスクリーンショットを使って質問できます。いちいちファイルに名前をつけて保存する手間は不要です。

1. `Win + Shift + S` で画面の必要な部分を切り取る（クリップボードにコピーされる）
2. AIに向かって「今のクリップボードの画像を解説して」「この画像を見て」と指示する
3. AIが裏側で自動的に抽出ツール（`tools/get_clipboard.py`）を起動し、画像を読み込んで解説してくれます。
※一時ファイルはAIが自動で破棄するため、ゴミデータは一切残りません。

### 3. ナレッジグラフ（キーワードリンク）の更新
ノートが増えてきたら、AIに「キーワードをリンク化して」と頼むだけで、全てのドキュメントの重要単語（「MVC」「参照渡し」など）が自動的に `[[キーワード]]` の形に変換されます。
これをObsidianで開くことで、関連知識が蜘蛛の巣状に可視化されます。

## システム構成（AIエージェントとの連携）

このディレクトリには、AIエージェントが「人間の手を煩わせることなく自律的に動く」ための仕組みが用意されています。

* **`AI_INSTRUCTIONS.md`**
  AIエージェントに読ませるための「絶対遵守ルール（取扱説明書）」です。有能なAIは起動時にこれを読み込み、「このディレクトリでの振る舞い方（画像の読み方やファイルの作り方）」を自律的に理解します。

* **`tools/` フォルダ**
  AIが実行するための裏方スクリプト群が格納されています。
  * `get_clipboard.py`: クリップボードから画像を抽出し、AIが読める一時ファイルに変換するツール。
  * `link_keywords.py`: ディレクトリ内の全Markdownファイルを走査し、キーワードをObsidianリンク化するツール。

この仕組みにより、他のCLIツールや新しいAIエージェントを導入した際にも、シームレスな体験が引き継がれます。
