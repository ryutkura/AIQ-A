# 自宅でのLinux実機学習とRaspberry Piの選び方

「実機ベースでLinuxを触ってみたい」というモチベーション、素晴らしいです！仮想環境（DockerやWSLなど）とは違い、ネットワーク構成やOSの起動プロセスを肌で感じられるため、インフラの学習効果は段違いです。

## 1. Raspberry Piはベストな選択肢か？
結論から言うと、**「非常に優れたベストな選択肢の1つ」**です。
* **メリット**: インターネット上の情報量が圧倒的に多い（困ってもググればすぐ解決する）、省電力で24時間稼働させやすい、GPIOピンを使って電子工作にも拡張できる。
* **推奨[[モデル]]**: Linuxの学習（Webサーバー構築やDockerなど）を行うなら、**Raspberry Pi 4 [[Model]] B (メモリ4GB以上)** または最新の **Raspberry Pi 5** が必要です。

## 2. 【重要】Raspberry Pi Pico W でLinuxは動く？
**結論：Raspberry Pi Pico (および Pico W) では、Linuxは動きません。**

名前は同じ「Raspberry Pi」ですが、両者は「パソコン」と「電卓」くらい根本的に種類が違います。

* **Raspberry Pi 4 / 5 (シングルボードコンピュータ)**:
  スマホやパソコンと同じ仕組みを持っており、Linuxという巨大な「OS」を動かすことができます。私たちがターミナルでログインしてコマンドを叩けるのはこちらです。
* **Raspberry Pi Pico シリーズ (マイクロコントローラ / マイコン)**:
  OSを持たず、書き込まれた1つのプログラム（Micro[[Python]]や[[C言語]]のコード）だけをひたすらループ実行する「部品」です。温度を測る、モーターを回すといった単一の仕事に特化しています。Pico WはWi-Fiモジュール（W）が付いているため通信はできますが、ターミナルでログインしてLinuxのコマンドを叩くような使い方はできません。

## 3. 実機学習の他の選択肢
もし「純粋にLinuxサーバー構築の練習をしたい（電子工作はしない）」のであれば、ラズパイ以外の選択肢も非常に強力です。現在、ラズパイは本体以外の周辺機器（電源、ケース、SDカードなど）を揃えると1.5万円〜2万円近くになってしまうためです。

1. **中古のノートPCやデスクトップPC**:
   実は**これが一番コスパ最強**です。家に眠っている古いWindows PCがあれば、USBメモリからUbuntu（Linuxの一種）などをインストールするだけで、立派なLinuxサーバーになります。ラズパイよりもはるかに高性能です。
2. **格安ミニPC (Intel N100プロセッサ搭載など)**:
   最近流行している2万円台の手のひらサイズのWindows PCを買い、Linuxを入れ直す方法です。ラズパイを一式揃えるのと価格が変わらない上に、性能は圧倒的に上です。

## まとめ
* 電子工作も視野に入れたい、ラズパイというハード自体を楽しみたい → **Raspberry Pi 4 または 5**
* 純粋なLinux（Webサーバー構築、Docker等）の学習に特化したい → **家に眠っている古いPC** や **格安ミニPC**
* ※ **Pico / Pico W** はIoT機器（センサーやスイッチ）を作るための「マイコン」なので、Linuxの学習には使えません。

## 4. 「LinuxもIoTも両方やりたい！」という場合の最強構成
「どっちも面白そうだから両方やりたい」という場合、学習のアプローチとして以下の2つの構成が考えられます。

### パターンA：全部入り構成（Raspberry Pi 4 / 5）
* **構成**: Raspberry Pi 4 または 5 を1台買う。
* **特徴**: 1台の機械の中で完結します。Linux上でWebサーバーを動かしながら、同じ基板上に生えている「GPIOピン」にセンサーを繋いで、[[Python]]で直接データを読み取ることができます。入門として非常に手軽で王道のスタイルです。

### パターンB：本格的IoTアーキテクチャ構成（PC ＋ Pico W）
* **構成**: 家の古いPC（または格安ミニPC） ＋ Raspberry Pi Pico W（約1,500円）
* **特徴**: 実はこれが**一番面白く、実務に近い構成**です。
  * **PC側**: どっしり構える「Linuxサーバー」として動かします。Dockerを使って[[データベース]]（DB）やデータ可視化ツールを構築し、IoT機器からのデータを受け止める「基地局（クラウド/オンプレ）」にします。
  * **Pico W側**: 部屋の隅や窓際に置く「末端のIoTデバイス（エッジ）」として動かします。センサーで温度や明るさを測り、Wi-Fiを使ってサーバー（PC）にデータを飛ばします。
  * **メリット**: 「Linuxサーバーの構築・運用」と「マイコンによるハードウェア制御と通信」の両方を、実際のIoTシステムと全く同じネットワーク構成で学べます。

もし家に使っていないPCがあるなら、無理に高いラズパイ本体を1つ買うよりも、「PCをLinux化」して「Pico Wを数個買って家中に置く（パターンB）」の方が、圧倒的に安上がりで、かつ本格的なIoTシステム構築を楽しめるのでオススメです！

### 「お古のPCがない」場合の現実的な選択肢
もし家に余っているPCがない場合、以下の3つの現実的なアプローチがあります。

1. **Raspberry Pi 4/5のスターターキットを買う (王道)**
   * **予算**: 15,000円〜25,000円
   * 本体、電源、ケースなどが全て揃ったキットを買うのが一番手っ取り早いです。最初は「全部入り」として使い、後からPico Wを買い足せば「ラズパイをサーバー機」「Picoを末端センサー」とする本格構成への移行も可能です。
2. **格安ミニPC（Intel N100搭載機など）を新規に買う**
   * **予算**: 20,000円〜25,000円前後
   * Amazonなどで売られている「手のひらサイズのWindows PC」を買い、Windowsを消してLinuxをインストールします。実はラズパイを一式揃えるのと価格がほぼ同じで、性能はラズパイより遥かに高いため、近年非常に人気のある選択肢です。
3. **クラウドのVPS（仮想サーバー）を借りる (実務IoTの完全再現)**
   * **予算**: 月額500円〜1,000円程度 ＋ Pico W本体（約1,500円）
   * さくらのVPSやConoHa VPSなどで「月額数百円のLinuxサーバー（クラウド）」を借りて、自宅のPico Wからインターネット経由でデータを飛ばします。手元にLinuxの物理マシンはありませんが、これこそが**現代のスマートホームや企業IoTと全く同じクラウドアーキテクチャ**です。ハードを買わないため初期費用が最も安く済みます。
