# Linuxのユーザー管理と権限（パーミッション）設計

ユーザーからの「rootでログインできたらやりたい放題じゃねーかよ」というツッコミは、まさにその通りであり、Linuxの権限設計の核心を突いています。

複数人が同時に同じサーバー（Linuxマシン）を利用するマルチユーザー環境において、システムが崩壊しないようにどのようなルールが敷かれているのかを解説します。

## 1. root（特権ユーザー）とsudoの概念
おっしゃる通り、**root（ルート）ユーザーは神様であり、文字通り「やりたい放題」です。** 他人のファイルを消すことも、システムを破壊するコマンドを実行することも一瞬で可能です。

* **原則 root ではログインしない**:
  その「やりたい放題」は、ちょっとしたタイポ（打ち間違い）でシステム全体を消し飛ばすリスク（有名な `rm -rf /` など）を伴います。そのため、現代の運用では「普段は一般ユーザーとしてログインする」のが鉄則です。
* **`sudo` コマンド**:
  どうしても神様の力が必要な時（ソフトのインストールや全体設定の変更など）だけ、一般ユーザーが一時的にrootの権限を借りる仕組みが `sudo` (Superuser DO) です。これを使うと「誰が、いつ、神の力を使ったか」がログに残るため、セキュリティと監査の面でも非常に重要です。

## 2. 権限（パーミッション）の仕組み
自分以外のユーザーが勝手に自分のファイルを書き換えたり実行したりできないように、ファイルやディレクトリには厳格なロックがかかっています。

Linuxのファイルは、以下の3つの対象に向けて権限を設定します。
1. **所有者 (User)**: そのファイルを作った人
2. **グループ (Group)**: プロジェクトチームなどのグループ
3. **その他 (Others)**: それ以外の全員

そして、それぞれに対して「3つの権利」を与えたり奪ったりします。
* **r (Read)**: 読み取り権限
* **w (Write)**: 書き込み権限
* **x (Execute)**: 実行権限

例えば「所有者は読み書きできるが、グループやその他の人は読み取りしかできない (`rw-r--r--`)」といった細かい制御を、`chmod`（権限変更）や `chown`（所有者変更）といったコマンドで行います。これにより、ユーザー間のデータコンフリクト（競合や破壊）を防ぎます。

## 3. 複数人作業時のプロセス優先度とリソース競合
複数人が同時に重い処理を実行した場合の交通整理は、Linuxの「カーネル（OSの核）」が自動で行っています。

* **プロセスのスケジューリング**:
  カーネルは、全ユーザーのプロセスに対して、ミリ秒単位で順番にCPUを割り当てて処理を進めます（タイムシェアリング）。
* **優先度（Nice値）**:
  プロセスには `-20` (最優先) から `19` (一番後回し) までの「Nice（ナイス）値」という優先度が設定されています。
  * **一般ユーザーの制限**: 一般ユーザーは、自分が実行するプロセスの優先度を「下げる（他の人にCPUを譲る＝よりNiceになる）」ことしかできません。
  * **rootの特権**: 優先度を「上げる（他のプロセスを差し置いて自分の処理を優先させる）」ことができるのは、root権限だけです。これにより、一人のユーザーがシステムリソースを独占することを防いでいます。

**まとめ**：
Linuxは「みんなで1つのコンピュータを共有する」ことを前提（マルチユーザー）に作られたOSです。そのため、「所有者と権限」によるデータの保護と、「カーネルによるCPUやメモリの公平な分配」、そして「神様(root)の力を無闇に使わせない仕組み(sudo)」によって、平和が保たれています。
