# [[C言語]]における引数渡しと[[参照外し]]

[[C言語]]には「[[参照渡し]]」という言語仕様は存在せず、**「すべて[[値渡し]] (Pass-by-Value)」**です。

## 1. [[ポインタ]]を使った「[[参照渡し]]」のシミュレーション
関数に「変数の[[アドレス]]（住所）」をコピーして渡すことで、呼び出し元の変数を操作します。

```c
void modify(int *p) {
    *p = 20; // 2. 「[[参照外し]]」をして中身を書き換える
}

int main() {
    int x = 10;
    modify(&x); // 1. xの「住所」をコピーして渡す
    // x は 20 になっている
}
```

## 2. 「[[参照渡し]]」と「[[参照外し]]」の混同に注意

用語は似ていますが、全く別のことを指します。

- **[[参照渡し]] (Pass-by-Reference)**: 概念。呼び出し元の変数を直接共有するような渡し方。
- **[[参照外し]] (Dereferencing)**: **具体的な操作**。[[ポインタ]]が指すメモリの[[アドレス]]へ実際に行って、データを取り出したり書き換えたりすること。[[C言語]]の `*p` という書き方のこと。

## 3. [[Java]]/[[Python]]との比較：明示か隠蔽か
[[C言語]]が他の言語と最も違うのは、**「参照（住所）を扱っていること」をコード上で隠さない**点です。

| 比較項目 | [[C言語]] | [[Java]] / [[Python]] |
| :--- | :--- | :--- |
| **渡し方の実態** | [[アドレス]]（数値）の[[値渡し]] | 参照の[[値渡し]] |
| **住所の取得** | `&x` と明示する | 自動で行われる |
| **[[参照外し]]** | `*p` と明示する | `obj.field` などで自動で行われる |

### [[Java]]/[[Python]]が「不思議」に見える理由
[[Java]]や[[Python]]では、この `&` や `*` に相当する操作を言語側が勝手にやってくれます。
そのため、「いつの間にか中身が変わっている（ように見える）」のですが、[[C言語]]ではすべてプログラマが記号を使って指示するため、**「何がコピーされ、どこで実体を見に行っているか」**が100%把握できるようになっています。

## まとめ
- [[C言語]]は「住所」という値を渡しているだけ。
- `*` を使って住所の先を見に行く操作が「[[参照外し]]」。
- [[Java]]/[[Python]]は、この[[C言語]]のような[[ポインタ]]操作を「裏側で自動的に」やっている言語だと言える。
