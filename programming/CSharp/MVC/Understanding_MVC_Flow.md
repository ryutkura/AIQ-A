# [[MVC]]における情報のやり取り（データフロー）

## 1. 「main関数から始まる世界」との決定的な違い
[[C言語]]や基本情報技術者試験などで学ぶプログラムは、`main()` 関数から始まり、上から下へ順番に処理が進みます（自分が全体のループを制御する）。

しかし、[[MVC]]のようなWebフレームワークでは**「制御の反転（Inversion of Control）」**が起きています。
つまり、**「あなたがフレームワークを呼び出す」のではなく、「フレームワークがあなたの書いたコード（[[コントローラー]]）を呼び出す」**のです。

（実はASP.NET Coreにも見えないところで `Program.cs` の中に `Main` 関数があり、そこでWebサーバーが起動して「リクエストを待つ無限ループ」に入っています。私たちはそのループの中の一部だけを書かされている状態です。）

## 2. レストランに例える[[MVC]]の情報のやり取り

この「情報のやり取り」はレストランをイメージすると非常に分かりやすいです。

*   **お客さん (ブラウザ):** 「ハンバーグ定食（[[URL]]のリクエスト）」を注文する人。
*   **ウェイター ([[Controller]]):** 注文を受け取り、厨房に指示を出し、出来上がった料理をお客さんに運ぶ「仲介役」。
*   **厨房・シェフ ([[Model]]):** ウェイターからの指示で、冷蔵庫（[[データベース]]）から肉を取り出し、調理（計算やデータ処理）を行う「裏方」。
*   **盛り付け担当 ([[View]]):** 厨房から上がってきた料理（データ）を、綺麗なお皿に盛り付けて見栄えを良くする（HTMLを作る）「デザイナー」。

## 3. 実際の情報の流れ（ステップ・バイ・ステップ）

お客さんがブラウザで `http://あなたのサイト/User/Profile/5`（ユーザーID 5番のプロフィール画面）にアクセスしたとします。

1.  **リクエストの到着 (ブラウザ → フレームワーク)**
    *   [[URL]]がサーバーに届きます。ここでフレームワークの**[[ルーティング]]**という機能が働き、「この[[URL]]なら `User[[Controller]]` の `Profile` という処理を呼び出そう、おまけの数字は `id=5` だな」と解析します。
2.  **[[Controller]]の呼び出し (フレームワーク → [[Controller]])**
    *   フレームワークがあなたの書いた[[コントローラー]]を呼び出します。
    *   `public IActionResult Profile(int id)` のようなメソッドが呼ばれ、引数 `id` に `5` が渡されます。
3.  **[[Controller]]から[[Model]]への依頼 ([[Controller]] → [[Model]])**
    *   ウェイター（[[Controller]]）は厨房に依頼します。
    *   `var userData = [[データベース]]からユーザーを探す処理(id);`
4.  **[[Model]]から[[Controller]]への返却 ([[Model]] → [[Controller]])**
    *   厨房（[[Model]]）は[[データベース]]から名前や年齢などのデータを取り出し、[[Controller]]に渡します。
5.  **[[Controller]]から[[View]]へのデータの受け渡し ([[Controller]] → [[View]])**
    *   [[Controller]]は受け取ったデータを[[View]]（盛り付け担当）に渡します。
    *   コードでは `return [[View]](userData);` のように書きます。ここで**情報（userData）が[[View]]に移動**します。
6.  **HTMLの生成と返却 ([[View]] → ブラウザ)**
    *   [[View]]は渡された `userData` を使って、「こんにちは、〇〇さん！」といったHTML（画面の設計図）を作成し、最終的にお客さんのブラウザに返します。

## 4. フレームワーク嫌いを克服する第一歩

最初は「裏で勝手に何かが動いている（ブラックボックス）」ように感じるため、気持ち悪く思うのが普通です。
まずは、**「リクエスト（[[URL]]） → [[Controller]] → [[Model]] → [[Controller]] → [[View]] → レスポンス（画面）」** という一本道のリレーが行われている、というイメージを持つところから始めてみてください。
