# [[参照渡し]] (Pass-by-Reference) について

## 1. 概要
[[参照渡し]]とは、プログラムにおいて関数やサブルーチンに引数を渡す際、変数の「値」をコピーして渡すのではなく、その変数が格納されているメモリ上の「参照（[[アドレス]]情報）」を渡す方式のことです。

## 2. 特徴
- **呼び出し元への影響:** 関数内部で引数の値を変更すると、呼び出し側の元の変数の値も書き換わります。
- **効率:** 大規模なオブジェクトや構造体を渡す場合、コピーを作成しないため、メモリ消費を抑え、処理を高速化できる利点があります。

## 3. [[値渡し]]との比較

| 項目 | [[値渡し]] (Pass-by-Value) | [[参照渡し]] (Pass-by-Reference) |
| :--- | :--- | :--- |
| **渡すもの** | データのコピー | データのメモリ上の場所 (参照) |
| **関数内での変更** | 元の変数には影響しない | 元の変数も書き換わる |
| **主な用途** | 基本データ型、値を変えたくない場合 | 大きなデータ構造、関数内で値を更新したい場合 |

## 4. 各言語における扱い（詳細）

### [[Java]] / [[Python]] における「参照の[[値渡し]]」
[[Java]]や[[Python]]では、厳密には「[[参照渡し]]」ではなく**「参照の[[値渡し]]（Pass-by-Object-Reference / Pass-by-Sharing）」**と呼ばれます。

- **参照の[[値渡し]]:** 「オブジェクトへの参照（メモリ上の住所）」という情報を、値としてコピーして渡します。
- **挙動の違い:**
    - **オブジェクトの操作:** 渡された参照を通じてオブジェクトの中身（リストの要素やインスタンスのフィールド）を書き換えると、呼び出し元にも反映されます。
    - **変数の再代入:** 関数内で引数そのものに別のオブジェクトを代入しても、呼び出し元の変数が指す先は変わりません。
