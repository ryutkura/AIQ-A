# 参照渡し (Pass-by-Reference) について

## 1. 概要
参照渡しとは、プログラムにおいて関数やサブルーチンに引数を渡す際、変数の「値」をコピーして渡すのではなく、その変数が格納されているメモリ上の「参照（アドレス情報）」を渡す方式のことです。

## 2. 特徴
- **呼び出し元への影響:** 関数内部で引数の値を変更すると、呼び出し側の元の変数の値も書き換わります。
- **効率:** 大規模なオブジェクトや構造体を渡す場合、コピーを作成しないため、メモリ消費を抑え、処理を高速化できる利点があります。

## 3. 値渡しとの比較

| 項目 | 値渡し (Pass-by-Value) | 参照渡し (Pass-by-Reference) |
| :--- | :--- | :--- |
| **渡すもの** | データのコピー | データのメモリ上の場所 (参照) |
| **関数内での変更** | 元の変数には影響しない | 元の変数も書き換わる |
| **主な用途** | 基本データ型、値を変えたくない場合 | 大きなデータ構造、関数内で値を更新したい場合 |

## 4. 各言語における扱い（補足）
- **C言語:** 厳密にはすべて値渡しですが、ポインタ（アドレス値）を値として渡すことで、参照渡しと同等の動作をシミュレートします。
- **Java / Python:** プリミティブ型以外は「参照」を扱いますが、その「参照という値」を渡している（Pass-by-Object-Reference）ため、言語仕様上の定義は厳密には異なりますが、ミュータブル（変更可能）なオブジェクトの場合は参照渡しに近い挙動を示します。
