# CLI環境におけるクリップボード画像のAI解析メカニズム

CLI（コマンドラインインターフェース）は本来テキストしか扱えませんが、AIエージェントに「クリップボードの画像を読ませる」ことで、ファイルを一切保存・管理することなくシームレスに画像認識を行わせる画期的なワークフローが存在します。

このドキュメントでは、その裏側で何が起きているのか（仕組み）を解説します。

## ワークフローの全体像

ユーザーが「画像の解説をお願い」と頼んでから、AIが回答を返すまでの数秒間で、システムとAIは以下の連携を行っています。

### 1. クリップボードへの画像格納（ユーザー操作）
ユーザーが `Win + Shift + S` などを使い画面をスクショすると、WindowsのOSは画像データをファイルとしてではなく、メモリ上の「[[クリップボード]]」領域に一時的に保持します。

### 2. クリップボードの抜き出し（AIによる自動実行）
AIはユーザーからの指示を受けると、CLIの裏側で[[Python]]スクリプトを自動実行します。
具体的には、画像処理ライブラリである `Pillow` を呼び出し、OSのクリップボードAPIにアクセスします。

```python
from PIL import ImageGrab

# クリップボードのデータを取得
img = ImageGrab.grabclipboard()

if img:
    # データが存在すれば、一時ファイルとして書き出す
    img.save('_temp_clipboard.png')
```
この処理により、メモリ上にしかなかった画像が、一瞬だけ実体のある画像ファイル（`_temp_clipboard.png`）としてカレントディレクトリに生成されます。

### 3. 画像の読み込みと解析（AIの視覚[[モデル]]稼働）
AIは、生成された一時ファイル `_temp_clipboard.png` を自分自身のファイル閲覧機能（ツール）を使って読み込みます。読み込んだ画像データはAIの「マルチモーダル（視覚）[[モデル]]」に渡され、テキストと一緒に解析が行われます。

### 4. 証拠隠滅・自動クリーンアップ（AIによる自動実行）
画像を読み取って内容を理解した直後、AIは回答を生成するのと並行して、OSのファイル削除コマンド（Windowsの場合は `del _temp_clipboard.png`）を実行します。

これにより、一時ファイルは完全に消去され、ユーザーのPC内には「名前をつけるのに困るスクショファイル」といったゴミデータが一切残りません。

## この仕組みのメリット

* **ファイル管理が不要**: 保存ダイアログを開き、名前をつけて保存する手間（カチャカチャする面倒さ）がゼロになります。
* **AIのすれ違い（誤読）防止**: 一時ファイルは常に1つだけ生成され、即座に破壊されるため、過去の画像をAIが誤って読み込む事故が構造上発生しません。
* **思考の邪魔をしない**: ユーザーは「撮る」「投げる」だけでよいため、プログラミングや数学の学習という本来の目的に100%集中できます。
