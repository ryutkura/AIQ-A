# [[Python]]における引数渡し

[[Python]]の引数渡しは**「オブジェクト[[参照渡し]] (Pass-by-Object-Reference)」**、あるいは**「代入による渡し (Pass-by-Assignment)」**と呼ばれます。

## 1. ミュータブル（変更可能）なオブジェクト
[[リスト]] (`list`)、[[辞書]] (`dict`)、集合 (`set`) など。
[[関数]]内での変更が呼び出し元に反映されます。

### 具体例
```python
def modify_list(lst):
    lst.append(4)  # 元の[[リスト]]を操作

def reassign_list(lst):
    lst = [10, 20] # 新しい[[リスト]]を代入（呼び出し元には影響しない）

my_list = [1, 2, 3]
modify_list(my_list)
print(my_list)    # 結果: [1, 2, 3, 4]

reassign_list(my_list)
print(my_list)    # 結果: [1, 2, 3, 4] (再代入の影響はない)
```

## 2. イミュータブル（変更不能）なオブジェクト
数値 (`int`, `float`)、文字列 (`str`)、タプル (`tuple`) など。

### 「名札（ラベル）」の貼り直し
[[Python]]の[[変数]]は箱ではなく、オブジェクトに貼り付けた**「名札」**です。
`s += " world"` という記述は、内部的には `s = s + " world"` と同じです。

1. `s + " world"` という**新しいオブジェクト**がメモリに作られる。
2. [[関数]]内の名札 `s` を、古いオブジェクトから剥がして**新しい方に貼り直す**。
3. 呼び出し元の名札は「古いオブジェクト」に貼られたままなので、何も変わりません。

## 3. 値を変更したい（反映させたい）場合のパターン

### パターンA: 戻り値で受け取る（推奨）
イミュータブルな値を扱う際の最も標準的な方法です。
```python
def add_suffix(s):
    return s + "_updated"

name = "data"
name = add_suffix(name) # 外側で名札を貼り直す
```

### パターンB: ミュータブルな容器に入れる
どうしても直接書き換えたい場合は、[[リスト]]や自作[[クラス]]などの「容器」に入れます。
```python
def update_first_element(container):
    container[0] = "new_value" # 容器の中身を入れ替える

my_data = ["old_value"]
update_first_element(my_data)
print(my_data[0]) # "new_value"
```

## まとめ
- **ミュータブル**: 名札が指している「モノ」の中身をいじる（外に影響する）。
- **イミュータブル**: 新しい「モノ」を作って、[[関数]]内の名札だけ貼り直す（外に影響しない）。
- **再代入 (`=`)**: 常に名札の「貼り直し」なので、外の名札には影響しない。
